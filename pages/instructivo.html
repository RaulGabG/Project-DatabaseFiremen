Instructivo para uso de la Base de Datos




Acceso

Desde la computadora principal (donde está instalada la base de datos) solamente es necesario ingresar desde un navegador a la dirección: localhost/nombeDeLaBaseDeDatos

Desde otro dispositivo conectado a la misma red de internet se tiene que ingresar a la dirección: 192.168.100.numeroLocal/nombreDeLaBaseDeDatos

Esto da acceso a la página de inicio de sesión, el usuario y contraseña es único y será provisto por el administrador en caso de ser necesario.

Las carpetas con los escaneos de los legajos se encuentran alojadas en Google Drive: [mail o dirección de Google Drive]

La contraseña de acceso será otorgada también por el administrador.

Es fundamental para la seguridad de la información que una vez se haya terminado el trabajo o consulta en la base, se cierre la sesión, para lo cual se encuentra un botón en la barra de navegación y también en partes estratégicas de la página.

Se recomienda abrir las mismas en una pestaña de navegación en privado, o no guardar los datos de inicio de sesión en su defecto.


Sobre el uso

La interfaz web (página local, o página desde ahora) está pensada para simplificar la consulta, carga, modificación y borrado de datos en la base de datos, la cual originalmente se realiza con un lenguaje de programación que se llama SQL, no es necesario que se aprenda el mismo, pero sí prestar atención a las indicaciones en cada una de las páginas (marcas de agua en los formularios) y este instructivo. Principalmente:

  • La carga de formularios no debe tener datos vacíos, en el caso de no tener datos se utiliza la palabra NULL.
  • Aunque en la página de carga de datos se visualicen todos los formularios, se deben cargar de a uno a la vez, ya que cada formulario usa una configuración diferente para la carga de los mismos.
  • No se debe hacer uso de las comillas simples ( ‘ ) ya que son usadas por la programación de carga, haciendo que se generen errores internos.
  • El formato de fecha se escribe AAAA-MM-DD (año, mes, día). Esto luego al exportar los datos se interpreta directamente por Excel, u hoja de cálculo que se utilice.
  • Para modificar datos de debe saber el ID del mismo, en consulta de legajo se puede ver debajo de cada tarjeta el ID de modificación, de no tener este campo, se utiliza el número de legajo. Agregar datos a un legajo ya cargado cuenta como modificación (aunque el dato aparezca vacío en la consulta, tiene por defecto el contenido NULL).
  • La modificación requiere que ponga el nombre exacto de la tabla (ej: domicilio, legajo_personal), y de la columna (ej: apellido, fecha_nacimiento). Para lo cual se dejará una página de consulta especial con estas referencias en la página de modificación, y también en forma de anexo al final de esta página (igualmente cargar mal la referencia no dañará la base de datos, solamente no se realizará ninguna acción, teniendo que repetir la carga).
  • Como los nombres de tablas y columnas siguen una convención de no dejar espacios en blanco, sino que se unen mediante guión bajo ( _ ) es posible seleccionar fácilmente los mismos haciendo doble click sobre ellos, ya que los toma como una sola unidad.
  • La página de borrado de datos no tiene una opción de recuperación. Una vez que los datos son borrados, los mismos tiene que ser cargados de forma completa nuevamente.

Páginas de carga


Inicio


Página de bienvenida con recomendaciones importantes y accesos directos.

Cargar legajo


Página principal de carga, posee el acceso para carga de todas las tablas en formatos de formulario.

Es importante no dejar campos vacíos. Si no se tiene el dato se debe escribir la palabra reservada NULL. Esto igualmente no se verá en las exportaciones, pero es fundamental para funcionamiento de una base de datos.

Las marcas de agua recuerdan el tipo de dato que debe ser colocado cuando varía de la combinación normal de letras, números y símbolos (recordar que no deben utilizarse las comillas simples [ ‘ ]). La recomendación es igualmente consultar el anexo o página de formatos.

Ver legajos


Página de acceso a los legajos cargados. De forma resumida muestra los datos personales, pudiendo consultar los datos completos de un legajo en específico o utilizar los accesos a descargas para verlos en un formato de tabla.

Buscar legajo


Página similar a la anterior, pero enfocada a la búsqueda de un legajo en particular.

Modificar datos cargados


En esta página se debe tener especial cuidado con el nombre de las tablas y columnas, como con el formato del nuevo dato. Es prácticamente obligatorio el uso con el anexo o página de formatos. Es fundamental siempre poner el Id de modificación, el mismo se debe consultar antes en la página de búsqueda de legajo, encontrándose señalado entre paréntesis al fina de cada tarjeta.

Descargas


Página enfocada en las descargas de planillas. Las mismas se exportan en formato cvs, que son fácilmente interpretados por las hojas de cálculo. Esta página es de mucha utilidad para generar archivos de control de datos cargados y faltantes.

Borrar datos


Página destinada a la eliminación de datos cargados. Idealmente es mejor modificar un dato en lugar de borrarlo, pero pueden existir casos donde den error o el dato cargado deba retirarse.

Para borrar un dato el funcionamiento es similar a la modificación, solo que requiere menos campos:



Borrar legajos

En circunstancias normales la opción de borrar un legajo no debería usarse, pero en determinado caso puede ser necesario.

El funcionamiento es el más simple, solo se coloca el legajo.

Cerrar sesión


Es importante para la seguridad y privacidad de la base de datos siempre cerrar la sesión al terminar de hacer uso.

Una forma de no guardar los datos de la sesión es ingresar por la navegación en privado.

Se puede cerrar la misma desde el botón destacado en la barra de navegación y también luego de cada acción.
